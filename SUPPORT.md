# Support

`l40s-llm-bench` is an early-stage open source scaffold, so support is best
effort and centered on reproducibility.

## Where to Ask

- Use a bug report for crashes, incorrect output, or broken documented steps.
- Use a benchmark result report for reproducibility questions about a specific
  run.
- Use a feature request for proposed benchmark scenarios, serving-stack support,
  metadata fields, or result-summary improvements.
- Use the security policy for private vulnerability reports or accidental
  secret exposure.

## What Helps

When asking for help, include:

- repository commit;
- operating system and Python version;
- exact command;
- relevant configuration;
- whether the run was dry-run, fake-server, or real serving;
- OpenAI-compatible serving stack and version, if applicable;
- hardware details, especially if using L40S or similar GPUs;
- a small redacted log excerpt or result artifact.

Please do not include secrets, private endpoints, or confidential benchmark
data.

## Maintainer Expectations

Maintainers may ask for a smaller reproduction before investigating. Public
support does not include guarantees for private infrastructure, proprietary
models, cloud quota, or performance tuning for a specific deployment.
