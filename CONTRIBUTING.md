# Contributing

Thanks for considering a contribution to `l40s-llm-bench`.

This project is an early-stage scaffold for reproducible LLM inference
benchmark experiments, with an initial focus on OpenAI-compatible serving and
workflows that can later support L40S measurements. It does not claim benchmark
leadership, broad adoption, or validated production results.

## Good First Contributions

Useful contributions include:

- clearer reproducibility notes for benchmark runs;
- small fixes to benchmark configuration examples;
- improvements to raw result schemas, summaries, or manifest metadata;
- documentation for OpenAI-compatible serving setups;
- careful issue reports from dry runs, fake-server checks, or real hardware
  experiments.

## Ground Rules

- Keep changes small and reviewable.
- Separate infrastructure, documentation, and benchmark logic changes when
  possible.
- Do not submit benchmark claims without raw logs, configuration, environment
  metadata, and exact commands.
- Avoid publishing secrets, private endpoints, proprietary model names, or
  non-public hardware details in issues, pull requests, or result artifacts.
- Be explicit about what was measured, what was simulated, and what remains
  unvalidated.

## Development Setup

Install the development dependencies:

```bash
python -m pip install -r requirements-dev.txt
```

Run the current verification path:

```bash
python scripts/bench_openai_compatible.py --dry-run
python scripts/summarize_results.py --input results/raw/dry_run.jsonl --output-dir results/tables
python -m pytest
```

The dry run writes synthetic records only. It does not contact a model server,
download a model, or use a GPU.

## Benchmark Result Contributions

For benchmark-result issues or pull requests, include:

- repository commit;
- exact commands;
- configuration file or inline parameters;
- serving stack and version;
- model identifier, if public and shareable;
- hardware and driver/runtime details;
- raw JSONL logs or a clear reason they cannot be shared;
- summary output and any manifest generated by the repository tools.

If a result used an L40S GPU, say whether the GPU was bare metal, virtualized,
shared, thermally constrained, power-limited, or otherwise atypical. If a run
used an OpenAI-compatible endpoint, identify the serving implementation and
whether streaming was enabled.

## Pull Request Checklist

Before opening a pull request, please check:

- The scope is focused and described clearly.
- Any user-visible behavior is covered by a command, test, or reproducibility
  note.
- Benchmark claims are tied to evidence artifacts.
- No secrets or private endpoints are included.
- The pull request template is completed honestly, including limitations.

## Conduct

By participating, you agree to follow the project Code of Conduct.
